Pros of treesitter integration:

Could show function-level changes, not just file-level
Could visualize "this function calls that function which got deleted"
Could show type dependencies ("this struct changed, here's everything that uses it")
Could identify "blast radius" of a change
Could show a call graph or dependency graph

Cons:

Significant complexity increase
treesitter parsing for every commit would be slow
Language-specific (would need grammars for each language)
Might be scope creep - the user's core problem is "find when shit broke and restore it"
Dependency graphs can get REALLY messy visually

Middle ground:

Maybe a 6th view that's like a "blast radius" or "impact" view
Show function-level granularity only in Focus view
Keep it optional/lazy-loaded
